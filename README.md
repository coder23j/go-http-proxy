# go-host-proxy

`go-host-proxy` 是一个轻量级 HTTP 反向代理。它根据请求 `Host` 的第一个域名前缀自动解析目标地址，并把请求转发到对应的本机端口或指定 IPv4 地址端口。

## 功能特性

- 根据域名前缀转发到本机端口，例如 `3000.example.com` → `127.0.0.1:3000`
- 支持在域名前缀中嵌入 IPv4 和端口，例如 `192-168-1-10-8080.example.com` → `192.168.1.10:8080`
- 支持 WebSocket 透传
- 自动校验端口范围 `1-65535`
- 自动校验 IPv4 每段范围 `0-255`
- 禁止转发回自身，避免代理循环

## 构建

```bash
go build -o go-host-proxy main.go
```

项目当前是单文件 Go 程序，不依赖第三方库。

## 启动

默认监听 `127.0.0.1:21842`：

```bash
./go-host-proxy
```

指定监听地址：

```bash
./go-host-proxy -listen 127.0.0.1:8080
```

监听所有网卡的指定端口：

```bash
./go-host-proxy -listen :8080
```

## 转发规则

程序读取请求的 `Host` 头，取第一个 `.` 前面的内容作为前缀，然后按以下规则解析目标。

### 本机端口转发

如果前缀中包含合法端口，则转发到 `127.0.0.1:<端口>`。

| 请求 Host | 转发目标 |
| --- | --- |
| `3000.example.com` | `http://127.0.0.1:3000` |
| `app8080.example.com` | `http://127.0.0.1:8080` |
| `api-9000.example.com` | `http://127.0.0.1:9000` |

### IPv4 地址端口转发

如果前缀中包含 `IPv4-端口` 格式，即 `a-b-c-d-port`，则转发到对应 IPv4 地址和端口。

| 请求 Host | 转发目标 |
| --- | --- |
| `192-168-1-10-8080.example.com` | `http://192.168.1.10:8080` |
| `10-0-0-2-3000.example.com` | `http://10.0.0.2:3000` |

如果 IPv4 地址段不在 `0-255` 范围内，请求会返回 `400 Bad Request`。

## WebSocket

`go-host-proxy` 使用 Go 标准库 `httputil.NewSingleHostReverseProxy`，支持 WebSocket 升级请求透传。

例如后端服务运行在 `127.0.0.1:3000`，WebSocket 路径为 `/ws`，代理监听 `:21842`，可以访问：

```text
ws://3000.example.com:21842/ws
```

当前代理只转发到 `http://` 上游，因此对应 WebSocket 是 `ws://`。如需对外提供 `wss://`，建议在前面增加 Nginx、Caddy、Cloudflare 等 TLS 终止层。

## 错误处理

- `400 Bad Request`：`Host` 为空、前缀为空、没有合法端口、IPv4 格式非法等
- `403 Forbidden`：请求会转发回代理自身时返回 `禁止自转发`
- `502 Bad Gateway`：目标服务连接失败或代理转发失败

## 安全注意事项

默认监听地址为 `127.0.0.1:21842`，只接受本机访问。

如果显式使用 `-listen :21842` 或 `-listen 0.0.0.0:21842`，服务会暴露到所有网卡。公网环境下请务必增加访问控制。

需要特别注意：IPv4 地址端口转发能力可能被用于访问内网服务。如果不希望暴露内网访问能力，请保持监听在 `127.0.0.1`，或者在前置代理、防火墙中限制来源 IP、域名和可访问端口。

## 示例

启动一个本地服务：

```bash
python3 -m http.server 3000
```

启动代理：

```bash
./go-host-proxy -listen 127.0.0.1:21842
```

通过指定 Host 访问：

```bash
curl -H 'Host: 3000.example.com' http://127.0.0.1:21842/
```

请求会被转发到：

```text
http://127.0.0.1:3000/
```
